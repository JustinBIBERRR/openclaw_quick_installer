# UX Refresh 验收测试计划

## 概述

本文档定义了 Installer UX Refresh 的验收标准和测试路径，确保新的用户体验在浏览器预览模式和 Tauri 真机模式下都能正常工作。

## 测试环境

### 浏览器预览模式
- 运行 `npm run dev`
- 在浏览器中访问 `http://localhost:5173`
- 模拟所有前端步骤和转场

### Tauri 真机模式
- 运行 `npm run tauri dev`
- 测试真实的安装、启动和恢复流程

## 核心验收路径

### 1. 全新安装流程（Happy Path）

**浏览器预览模式:**
- [ ] 页面加载后显示欢迎页
- [ ] 欢迎页显示 OpenClaw logo 和功能预览
- [ ] 点击"开始安装"进入系统检查页
- [ ] 系统检查页显示步骤条（4步）
- [ ] 模拟检查完成后进入安装页
- [ ] 安装页显示新的进度界面和日志区域
- [ ] 模拟安装完成后进入配置页
- [ ] 配置完成后进入启动页
- [ ] 模拟启动成功后进入成功页
- [ ] 成功页显示安装摘要和操作按钮
- [ ] 点击"进入管理控制台"进入 Manager

**Tauri 真机模式:**
- [ ] 应用启动后根据 manifest 状态正确进入对应步骤
- [ ] 系统检查能正确检测环境并处理权限提升
- [ ] 安装过程能正确执行并显示真实日志
- [ ] 配置页面能正确调用 CLI 命令
- [ ] 启动页面能正确启动 Gateway
- [ ] 成功页能正确显示真实的安装信息
- [ ] 从成功页能正确进入 Manager

### 2. 恢复流程测试

**场景 A: 安装中断恢复**
- [ ] 在安装过程中关闭应用
- [ ] 重新启动应用，应该直接进入安装页继续

**场景 B: 已完成安装恢复**
- [ ] 完整安装后关闭应用
- [ ] 重新启动应用，应该直接进入 Manager（跳过成功页）

**场景 C: 配置中断恢复**
- [ ] 在配置过程中关闭应用
- [ ] 重新启动应用，应该进入系统检查或安装页

### 3. 错误处理流程

**安装失败处理:**
- [ ] 模拟安装失败，显示错误信息和重试按钮
- [ ] 模拟 Node.js 下载失败，显示手动下载引导
- [ ] 重试功能能正确重新开始安装

**启动失败处理:**
- [ ] 模拟 Gateway 启动失败，显示诊断和修复选项
- [ ] 诊断功能能正确显示问题
- [ ] 修复功能能正确执行

### 4. 视觉和交互验证

**设计系统一致性:**
- [ ] 所有页面使用统一的背景色（#060B14）
- [ ] 玻璃面板效果正确显示
- [ ] 主色调（#00E5FF）在所有元素中一致
- [ ] 成功色（#39FF14）在成功状态中正确显示
- [ ] 圆角、间距、字体大小符合设计规范

**动效和过渡:**
- [ ] 步骤条进度动画流畅
- [ ] 按钮悬停效果正确
- [ ] 页面转场无闪烁
- [ ] Loading 动画正确显示

**响应式和窗口:**
- [ ] 窗口拖拽区域正确工作
- [ ] 窗口控制按钮（最小化、最大化、关闭）正确工作
- [ ] 内容在不同窗口大小下正确显示
- [ ] 滚动区域正确工作

### 5. 功能完整性验证

**步骤条:**
- [ ] 在欢迎页和成功页不显示步骤条
- [ ] 在其他页面正确显示4步进度
- [ ] 当前步骤高亮正确
- [ ] 已完成步骤标记正确

**日志系统:**
- [ ] 安装日志正确显示
- [ ] 启动日志正确显示
- [ ] 日志滚动和格式正确

**配置集成:**
- [ ] 配置页面正确集成到新的视觉系统
- [ ] 配置结果正确传递到成功页

## 已知限制和注意事项

1. **动效库依赖**: 当前使用 CSS 动画，如需更复杂动效可考虑引入 motion 库
2. **主题切换**: 当前只支持深色主题，未来可扩展主题系统
3. **国际化**: 当前只支持中文，未来可添加多语言支持
