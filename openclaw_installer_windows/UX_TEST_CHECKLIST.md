# OpenClaw Installer UX 回归清单

## 目标
验证当前“现代简洁清新”改版在核心用户路径上可用、可理解、可恢复。

## 测试环境建议
- Windows 10/11
- 干净账号（首次安装）与已安装账号（复访）各一套
- 网络正常与网络异常（可断网/代理）各测一轮

## 路径 1：首次安装（全流程）
- [ ] 打开安装器，进入向导 4 步（预检 -> 安装 -> 综合配置 -> 启动）
- [ ] 步骤条视觉状态清晰：当前步骤、已完成步骤可一眼识别
- [ ] 安装成功后进入综合配置，默认文案与主按钮可理解
- [ ] 配置 API + channel + skills/hook 后执行成功，进入启动页
- [ ] 启动 Gateway 成功后可打开 Chat（Web）

## 路径 2：已安装且未配置（直达 Manager）
- [ ] 打开 EXE 后直接进入 Manager（不再回到向导）
- [ ] 综合配置面板自动弹出
- [ ] Hero 区提示“建议先完成综合配置”文案正确
- [ ] “先完成综合配置”按钮视觉高亮且可直接完成配置
- [ ] 配置后关闭面板，状态卡 API 状态更新为“已配置”

## 路径 3：已安装且已配置（直达 Manager + 快速启动）
- [ ] 打开 EXE 后直接进入 Manager
- [ ] 顶部 Hero 展示 Chat 地址、API 状态、安装目录
- [ ] “启动并打开 Chat”一键可用
- [ ] 未启动状态下“启动并打开 Chat / 仅启动”按钮层级清晰
- [ ] 运行中状态下“打开 Chat / 重启 / 停止”操作符合预期

## 路径 4：飞书配置与连通性校验
- [ ] 在综合配置中选择 `channel=feishu` 后展示 `App ID/App Secret`
- [ ] 点击“校验飞书连通性”，成功时显示通过状态
- [ ] 输入错误密钥时，展示明确失败信息且不误导继续
- [ ] 保存后重新打开综合配置，飞书配置可回填

## 异常与恢复
- [ ] Gateway 启动失败时，错误卡片信息可读且有下一步动作
- [ ] “诊断与修复”可运行，结果面板可理解
- [ ] 日志入口（installer/运行日志）可正常打开
- [ ] 危险操作“卸载 OpenClaw”有二次确认，误触风险低

## 视觉一致性检查
- [ ] 主按钮/次按钮/危险按钮高度、圆角、字重一致
- [ ] 卡片分组（常用/维护/危险）边界清楚，不拥挤
- [ ] 深色背景下文字对比满足可读性（尤其灰色次级文案）
- [ ] hover/disabled/loading 状态在所有关键按钮表现一致

## 验收标准（建议）
- P0 路径（2/3）100% 通过
- P1 路径（1/4）>= 90% 通过
- 无阻断性问题（无法进入 Manager、无法启动/打开 Chat、无法保存配置）
