# OpenClaw Installer 手工验收脚本

本文档提供三套验收脚本，覆盖新装、升级和故障恢复三类场景。

---

## 前置准备

1. **下载最新现网包**：从 GitHub Release 下载 `openclaw-installer.exe`（单文件版）
2. **准备测试环境**：建议使用干净的 Windows 10/11 测试虚拟机或独立测试目录
3. **准备测试数据**：
   - API Key（Anthropic / OpenAI）
   - Base URL（如果需要测试自定义提供商）

---

## 验收脚本 1：新用户全新安装

**目标**：模拟全新 Windows 用户首次安装 OpenClaw 的完整流程。

### 前置条件

- 系统无 Node.js（或版本 < 18.x）
- 系统无 OpenClaw CLI
- `~/.openclaw/` 目录不存在

### 操作步骤

| 步骤 | 操作 | 预期结果 | 验收标准 | 结果 ✓/✗ |
|------|------|----------|----------|---------|
| 1 | 双击 `openclaw-installer.exe` | 弹出 GUI 窗口，显示"系统预检"页面 | - 窗口可正常拖动<br>- 右上角三按钮（最小化/最大化/关闭）可用<br>- 中文正常显示（无"?"乱码） | |
| 2 | 查看管理员权限检测 | 显示"❌ 否 - 需要管理员权限运行安装程序" | 红色提示文字清晰可见 | |
| 3 | 点击"授权管理员" | 弹出 UAC 提示框 | 选择"是"后，原窗口自动关闭，新窗口以管理员权限启动 | |
| 4 | 再次查看管理员权限检测 | 显示"✓ 是" | 绿色勾标记 | |
| 5 | 查看 Node.js 版本检测 | 显示"❌ 未安装或版本过低" | 红色提示 | |
| 6 | 查看 WebView2 检测 | 显示"✓ 已安装" | 绿色勾标记（Windows 10 1803+ 预装） | |
| 7 | 查看磁盘空间检测 | 显示"✓ 足够（XX GB）" | 绿色勾标记，实际可用空间显示正确 | |
| 8 | 点击"下一步" | 进入"安装 OpenClaw CLI"页面 | 页面自动开始安装流程 | |
| 9 | 等待 Node.js 安装 | 日志滚动显示"正在下载 Node.js..."、"正在解压 Node.js..." | - 进度百分比实时更新<br>- 无脚本错误弹窗<br>- 日志末尾显示"✓ Node.js 18.x 已安装" | |
| 10 | 等待 npm 全局安装 OpenClaw CLI | 日志显示"正在全局安装 openclaw..." | - 日志实时滚动<br>- 无 npm 错误<br>- 日志末尾显示"✓ OpenClaw CLI 已安装" | |
| 11 | 等待 CLI 版本验证 | 日志显示"openclaw version x.x.x" | 版本号正确显示 | |
| 12 | 安装成功 | 页面显示"✓ 安装成功！"，自动跳转到"配置 API Key"页面 | - 无红色错误提示<br>- 按钮"下一步"可用 | |
| 13 | 在"配置 API Key"页面输入 API Key | 选择"Anthropic"，输入有效 API Key | - 下拉菜单选项正确（Anthropic / OpenAI / Custom）<br>- 输入框可正常输入 | |
| 14 | 点击"下一步" | 进入"启动 Gateway"页面 | 页面显示"正在启动 OpenClaw Gateway..." | |
| 15 | 等待 Gateway 启动 | 日志滚动显示"Gateway starting on port 18789..." | - 10 秒内日志显示"Gateway started successfully"<br>- 浏览器自动打开 `http://localhost:18789/chat`<br>- 页面显示"✓ Gateway 已启动！" | |
| 16 | 在浏览器中测试 OpenClaw Chat | 输入"Hello"发送 | AI 正常回复，无"No API key found"错误 | |
| 17 | 返回 Installer，点击"完成" | 窗口自动跳转到"OpenClaw Manager"页面 | - 显示"Gateway 状态：运行中"<br>- 显示"端口：18789" | |
| 18 | 检查桌面快捷方式 | 桌面存在"OpenClaw"快捷方式，图标为红色龙虾 | - 图标正确<br>- 双击快捷方式直接进入 Manager 页面 | |
| 19 | 检查配置文件 | 打开 `~/.openclaw/openclaw.json` | - 文件存在<br>- 包含完整结构（`wizard`, `models`, `agents`, `messages`, `commands`, `session`, `gateway`, `meta`）<br>- `models.providers[0].key` 已正确填充 | |
| 20 | 关闭 Installer 和浏览器 | 关闭所有窗口 | Gateway 进程在后台继续运行（可用任务管理器验证） | |

**验收基线**：步骤 1-20 全部通过，无脚本错误，无配置缺失。

## 验收脚本 2：已有用户升级安装

**目标**：验证已安装 OpenClaw 的用户执行 Installer 的智能检测和跳过逻辑。

### 前置条件

- 系统已安装 Node.js 18+
- 系统已全局安装 `openclaw` CLI
- `~/.openclaw/openclaw.json` 已存在且配置有效

### 操作步骤

| 步骤 | 操作 | 预期结果 | 验收标准 | 结果 ✓/✗ |
|------|------|----------|----------|---------|
| 1 | 双击 `openclaw-installer.exe` | 弹出 GUI 窗口 | 窗口正常打开 | |
| 2 | 查看"系统预检"页面 | 显示"✓ Node.js 已安装（版本 18.x+）"<br>"✓ OpenClaw CLI 已安装" | 所有检测项为绿色勾标记 | |
| 3 | 点击"下一步" | **直接跳过"安装 OpenClaw CLI"页面**，进入"配置 API Key"页面 | - 无安装步骤<br>- 日志显示"检测到 OpenClaw CLI 已安装，跳过安装" | |
| 4 | 查看"配置 API Key"页面 | 显示已保存的 API Key（部分隐藏，如"sk-ant-***abc"） | - 提供商名称正确（Anthropic / OpenAI）<br>- API Key 前缀正确 | |
| 5 | 不修改配置，点击"下一步" | 进入"启动 Gateway"页面 | 页面显示"正在启动 OpenClaw Gateway..." | |
| 6 | 等待 Gateway 启动 | Gateway 正常启动，浏览器自动打开 | 无错误，流程与新装相同 | |
| 7 | 点击"完成" | 进入 Manager 页面 | 显示"Gateway 状态：运行中" | |
| 8 | 点击 Manager 页面的"修改 API Key" | 页面跳转回"配置 API Key"页面 | - 原 API Key 显示在输入框<br>- 可正常编辑 | |
| 9 | 修改 API Key 为新值，点击"保存" | 弹出提示"API Key 已保存" | - `~/.openclaw/openclaw.json` 中 `models.providers[0].key` 更新为新值<br>- 页面返回 Manager | |
| 10 | 点击"重启 Gateway" | Gateway 进程重启 | - 原进程被 `taskkill` 终止<br>- 新进程启动成功<br>- 浏览器刷新后 Chat 界面正常 | |

**验收基线**：步骤 1-10 全部通过，智能检测逻辑正确，无重复安装，配置修改生效。

## 验收脚本 3：故障恢复流程

**目标**：验证 Gateway 启动失败时的诊断、修复、重试流程。

### 前置条件

- 系统已安装 Node.js 18+ 和 OpenClaw CLI
- **人为制造故障**：删除或破坏 `~/.openclaw/openclaw.json`（例如删除 `gateway.auth.token`）

### 操作步骤

| 步骤 | 操作 | 预期结果 | 验收标准 | 结果 ✓/✗ |
|------|------|----------|----------|---------|
| 1 | 双击 `openclaw-installer.exe` | 弹出 GUI 窗口 | 窗口正常打开 | |
| 2 | 完成"系统预检"和"配置 API Key"步骤 | 流程正常 | 无错误 | |
| 3 | 进入"启动 Gateway"页面，等待启动 | **Gateway 启动失败**，页面显示：<br>"❌ Gateway 启动失败"<br>"错误代码：CONFIG_ERROR"<br>"错误信息：Gateway auth token missing"<br>"建议：配置文件可能损坏，建议运行诊断" | - 错误信息清晰显示<br>- "诊断问题"和"查看日志"按钮可用 | |
| 4 | 点击"诊断问题" | 页面显示"正在运行 openclaw doctor..."<br>随后显示诊断结果：<br>"⚠️ gateway.auth.token 缺失"<br>"✓ 1 个问题可自动修复" | - 诊断结果面板展开<br>- 问题列表清晰可读 | |
| 5 | 点击"一键修复" | 页面显示"正在运行 openclaw doctor --fix..."<br>随后显示"✓ 修复成功！已创建 gateway.auth.token" | - 修复结果面板更新<br>- 提示用户点击"重试" | |
| 6 | 点击"重试" | 页面重新尝试启动 Gateway | - Gateway 启动成功<br>- 浏览器自动打开 Chat 界面<br>- 页面显示"✓ Gateway 已启动！" | |
| 7 | 检查 `~/.openclaw/openclaw.json` | `gateway.auth.token` 字段已恢复 | 配置文件结构完整 | |
| 8 | 点击"完成" | 进入 Manager 页面 | 显示"Gateway 状态：运行中" | |
| 9 | **人为再次制造故障**：用任务管理器手动终止 Gateway 进程 | Gateway 进程不存在 | 进程列表无 `openclaw` 相关进程 | |
| 10 | 在 Manager 页面点击"诊断与修复" | 页面显示诊断结果：<br>"⚠️ Gateway 进程未运行" | 诊断正确识别进程缺失 | |
| 11 | 点击"启动 Gateway" | Gateway 重新启动 | - 新进程启动成功<br>- Manager 状态更新为"运行中" | |
| 12 | 点击"打开 Chat 界面" | 浏览器打开 `http://localhost:18789/chat` | - 页面正常加载<br>- Chat 功能可用 | |
| 13 | 点击"查看日志" | 打开 `~/.openclaw/installer-logs/` 目录 | - 目录包含所有命令日志文件（timestamped）<br>- 日志文件可读，包含 stdout/stderr | |
| 14 | 点击"复制诊断信息" | 剪贴板包含完整诊断信息（JSON 格式） | 可粘贴到文本编辑器，内容完整 | |

**验收基线**：步骤 1-14 全部通过，故障诊断准确，自动修复有效，重试逻辑正确。

## 验收通过标准

满足以下条件视为验收通过：

1. **脚本 1（新装）**：步骤 1-20 全部通过
2. **脚本 2（升级）**：步骤 1-10 全部通过
3. **脚本 3（故障）**：步骤 1-14 全部通过
4. **无以下禁入问题**：
   - 脚本执行过程出现退出码 1 且无明确错误提示
   - 配置文件 `openclaw.json` 缺失必要字段
   - Gateway 启动失败但诊断按钮不可用
   - 日志文件丢失或无法访问
   - 中文乱码（显示为"?"）
   - 窗口无法拖动或按钮无响应

## 附录：常见故障排查速查表

| 现象 | 可能原因 | 检查位置 | 快速修复 |
|------|----------|----------|----------|
| 安装时脚本退出码 1 | Node.js 下载失败 / npm 全局安装失败 | `~/.openclaw/installer-logs/` | 检查网络代理，手动安装 Node.js |
| Gateway 启动失败："auth token missing" | `openclaw.json` 配置不完整 | `~/.openclaw/openclaw.json` | 点击"诊断问题"→"一键修复" |
| Chat 界面显示"No API key found" | `openclaw.json` 中 `models.providers` 为空 | `~/.openclaw/openclaw.json` | 返回 Manager →"修改 API Key"重新保存 |
| 窗口无法拖动 | Tauri 窗口配置错误 | `src-tauri/tauri.conf.json` | 检查 `decorations: false` 和 `data-tauri-drag-region` |
| 日志目录为空 | `write_command_log` 未正确调用 | `src-tauri/src/commands.rs` | 检查所有 `run_openclaw_cmd` 调用点 |
